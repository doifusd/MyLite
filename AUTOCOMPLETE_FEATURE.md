# SQL 自动补全和语法高亮功能

## 功能概述

本更新为 SQL 编辑器增加了以下功能：

### 1. 表名和列名自动补全
- **动态加载**：在连接建立并选择数据库后，自动加载当前数据库中的所有表名和列名
- **智能补全**：在编辑器中输入时，会显示相关的表名和列名建议
- **上下文信息**：每个补全条目都包含类型和来源信息
  - 表名：显示表中包含的列列表
  - 列名：显示该列所属的表

### 2. 增强的 SQL 语法高亮
- **内置支持**：Monaco Editor 已提供了强大的 SQL 语法高亮
- **自定义高亮规则**：添加了自定义主题以优化 SQL 关键字、操作符、字符串和数字的显示
- **颜色方案**：
  - SQL 关键字：蓝色（加粗）
  - 操作符：红色
  - 字符串：绿色
  - 数字：橙色
  - 注释：绿色（斜体）

### 3. 扩展的关键字和函数补全
新增了多个常用的 SQL 关键字和函数：
- **类型/结构**：表名、列名
- **关键字**：除了原有的 SQL 关键字，新增了多个复合关键字（如 `UNION ALL`, `LEFT JOIN` 等）
- **函数**：添加了更多的 SQL 聚合函数和字符串函数

### 4. 快速快捷键
- **Ctrl+Enter (Cmd+Enter on Mac)**：执行查询
- **Shift+Alt+F**：格式化 SQL 代码

## 实现细节

### 后端（Rust）

**新增 Tauri 命令**：
```rust
#[tauri::command]
pub async fn get_database_tables_and_columns(
    state: State<'_, Arc<AppState>>,
    connection_id: String,
    database: String,
) -> Result<DatabaseTablesInfo, String>
```

此命令会：
1. 连接到指定的数据库
2. 查询所有表名
3. 对于每个表，查询其所有列名
4. 返回结构化的数据供前端使用

**新增数据结构**（`models/schema.rs`）：
- `TableCompletionInfo`：表示单个表及其列名
- `DatabaseTablesInfo`：表示整个数据库的表和列信息

### 前端（React/TypeScript）

**SQLEditor 组件增强**：
1. **状态管理**：添加 `tablesInfo` 和 `loadingTables` 状态来存储和管理表信息
2. **自动获取**：在组件挂载和数据库变化时自动获取表和列信息
3. **动态补全**：在 Monaco Editor 的补全提供程序中动态生成表名和列名的补全条目
4. **UI 反馈**：在加载表信息时显示加载状态指示器

## 使用方法

1. **连接到数据库**：通过连接对话框选择一个数据库连接
2. **切换数据库**：在 SQL 编辑器中改变数据库时，表列表会自动更新
3. **使用自动补全**：
   - 开始输入 SQL 查询时，按 `Ctrl+Space` 或等待编辑器显示补全菜单
   - 从列表中选择表名或列名

## 技术栈

- **前端**：React + TypeScript + Monaco Editor
- **后端**：Tauri + Rust + SQLx
- **数据库查询**：information_schema（MySQL）

## 性能考虑

- 表信息仅在第一次打开数据库或切换数据库时加载
- 自动补全使用客户端内存中的数据，不需要每次都查询数据库
- 支持大数据库（数百个表）的快速补全
